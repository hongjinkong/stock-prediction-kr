# 📈 주식 예측에서 리스크 관리까지 — 정직한 퀀트(Quant) 여정

> **AI로 주가를 예측하려다 "예측은 사실상 불가능하다"를 스스로 증명하고,
> 대신 실제로 작동하는 리스크 관리 시스템을 만들어 자동 페이퍼 매매까지 구현한 기록.**

---

## ⏱️ 30초 요약 (아무것도 몰라도 이해되는 버전)

- 🎯 **처음 목표**: AI로 미국 주식이 내일 오를지 내릴지 맞히기
- 🔍 **발견**: 엄격하게 검증하니 **맞힐 수 없었다.** 흔히 보이는 "고성능 백테스트"는 대부분 **데이터 누출·과적합**이 만든 착시였다. 누출을 제거하면 정확도가 동전던지기(50%) 수준으로 내려앉는다 → *시장은 그만큼 효율적이다.*
- 🔄 **전환**: 예측(맞히기)을 포기하고 **추세추종 리스크 관리**로 방향을 틀었다. 이건 실제로 작동한다 — 큰 손실(낙폭)을 막아준다.
- ✅ **결과**: **[Alpaca](https://alpaca.markets) 페이퍼(모의) 계좌에서 자동으로 매매하는 시스템**을 완성했다.
- 💡 **교훈**: **정확한 예측 ≠ 돈.** 그리고 **엄격한 검증**이야말로 스스로를 속이지 않게 해주는 가장 강력한 무기다.

> ⚠️ **이 저장소는 투자 자문이 아니며, 실제 거래를 권장하지 않습니다.** 학습·연구 기록입니다.

## 📖 이 프로젝트가 뭔가요? (조금 더 풀어서)

주식으로 돈을 벌려면 "미래 가격을 맞히면 되지 않나?"라고 생각하기 쉽습니다.
그래서 이 프로젝트는 **AI/머신러닝으로 주가 방향을 예측**하는 것으로 시작했습니다.

하지만 예측 모델을 만들다 보면 **함정**에 빠집니다:
- **데이터 누출(look-ahead)**: 미래 정보가 몰래 학습에 섞여 백테스트가 좋아 보이는 착각
- **과적합**: 과거 데이터에만 맞춰져 실전에선 무너지는 모델

이 프로젝트의 진짜 가치는, 이런 함정을 **하나하나 제거하며 정직하게 검증**한 것입니다.
그 결과 "일봉(하루 단위) 가격·기술지표로는 대형주의 방향을 맞힐 수 없다"는 **정직한 결론**에 도달했고,
그 위에 **예측이 아니라 리스크 관리로 승부하는 시스템**을 세웠습니다.

## ✅ 실제로 작동하는 것: 추세추종 시스템

예측을 포기한 뒤 채택한 방식입니다. **미래를 맞히지 않습니다. 이미 벌어진 추세에 반응만 합니다.**

- **롱(매수) 또는 현금**: 가격이 200일 이동평균 위(상승추세)면 보유, 아래로 꺾이면 현금
- **다자산 분산**: 지수·주식·금·채권 여러 자산 중 추세가 살아있는 것만 보유
- **변동성 조절**: 변동성이 큰 자산엔 적게, 작은 자산엔 많이 배분

**왜 이게 정직한가**: 파라미터(200일선 등)가 수십 년 된 관습값이라 **과적합이 없습니다.**
**핵심 가치**: 더 많이 버는 게 아니라 **폭락에서 덜 잃고 오래 살아남는 것**(2020·2022 같은 하락장 방어).

> 백테스트 참고(2010~2026, 9자산): **Sharpe 1.31 · 최대낙폭 -15.7%** (단순 지수 보유 대비 낙폭이 훨씬 얕음).
> ⚠️ 과거 성과가 미래를 보장하지 않습니다.

## 🚀 시작하기 (설치 & 실행)

### 1. 설치
```bash
pip install -r requirements.txt
```

### 2. (선택) API 키 설정
페이퍼 매매·펀더멘털을 쓰려면 `.env.example`을 복사해 **`.env`** 로 만들고 키를 채웁니다.
```
ALPACA_API_KEY_ID=...          # alpaca.markets 페이퍼 계좌 키 (자동매매용)
ALPACA_API_SECRET_KEY=...
SEC_USER_AGENT=이름 이메일       # SEC EDGAR 접근용 (팩터 노트북용)
```
> 🔒 `.env`는 gitignore로 **절대 커밋되지 않습니다.** 실제 키를 `.env.example`에 넣지 마세요.

### 3. 실행 (프로젝트 루트 폴더에서)

| 하고 싶은 것 | 명령어 |
|---|---|
| 📋 오늘의 목표 비중·주문 리포트 | `python scripts/advisory.py --nav 10000` |
| 👀 페이퍼 매매 미리보기 | `python scripts/paper_trade.py` |
| ✅ 페이퍼 계좌에 실제 주문 | `python scripts/paper_trade.py --execute` |
| 📈 성과 추적 (vs 시장) | `python scripts/monitor.py` |

> 연구 노트북은 `notebooks/` 폴더에서 주피터/VS Code로 열어 실행합니다.

## 💡 핵심 교훈 (이 프로젝트의 진짜 결과물)

1. **데이터 누출을 제거하면 "고성능"의 대부분이 사라진다** — 엄격함이 자기기만을 막는 최고의 자산
2. **일봉 기술지표로 대형주 방향 예측 = 알파 없음** (시장이 그만큼 효율적)
3. **정확한 예측 ≠ 돈이 되는 예측** — 예측되는 것(변동성)조차 단순 방법이 이미 가치를 다 담고 있으면 무의미
4. **예측 실패 ≠ 프로젝트 실패** — 리스크 관리로 전환한 것이 정직하고 견고한 답
5. **진짜 엣지는 더 좋은 모델이 아니라 다른 데이터/문제**에 있다 (독점 데이터, 덜 붐비는 시장 등)
